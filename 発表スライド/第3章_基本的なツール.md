# 達人プログラマー
## 第3章 基本的なツール

---

## 毎日使うツール、ちゃんと使えてますか？

毎日使うものほど、意外と使いこなせていないことがある。

```
- エディタのショートカット → あまり知らない
- Gitの操作 → とりあえずadd, commit, push
- バグを発見 → ひたすら println デバッグ…
```

---

## 達人の道具を支える **3本柱**

1. **バージョン管理** ─ すべての変更を記録しよう
2. **エディタ習熟** ─ 1つを極めよう
3. **デバッグ思考** ─ ゴムのアヒルに相談しよう

### なぜこの3つ？

```
        毎日使う（習慣）
       ┌─────┴─────┐
  バージョン管理     エディタ習熟
  （記録する）       （速く動く）
       └─────┬─────┘
         デバッグ思考
         （正しく直す）
```

## プレーンテキストという哲学

### データは「人間が読める形」で持とう

| 形式 | 10年後 | 検索 | 差分 |
|------|--------|------|------|
| バイナリ | ⚠️ 読めないかも | ❌ | ❌ |
| プレーンテキスト | ✅ 必ず読める | ✅ | ✅ |

### 実例

- 設定ファイル → JSON / YAML（読める）
- ドキュメント → Markdown（読める）
- データ保存 → CSV / テキスト（読める）

プレーンテキストはUnixの哲学 ─ どんなツールとも連携できる

---

## バージョン管理 ─ すべての変更を記録しよう

### Gitは「コードのタイムマシン」

> コードだけじゃない。**すべて**をバージョン管理に入れよう。

```
✅ ソースコード        → 当然
✅ 設定ファイル        → もちろん
✅ ドキュメント        → 必ず
✅ ビルドスクリプト    → 忘れずに
✅ インフラ構成（IaC） → できれば
```

---

## エディタ習熟 ─ 1つを極めよう

### なぜ「1つを深く」なのか？
```
❌ 複数エディタをそこそこ使う
   → どのエディタでも「ちょっと手が止まる」

✅ 1つのエディタを完全に使いこなす
   → 操作が無意識になり、思考に集中できる
```

### 使いこなせている目安

- よく使う操作をキーボードだけで完結できる
- マクロ・スニペットを自分で作れる
- プラグインを入れてカスタマイズできる
- コーディング中にメニューを探していない

---

## デバッグの基本的な考え方

### まず疑うべきはどこ？

```
バグ発生！

❌ 「OSのバグかも…」
❌ 「コンパイラがおかしい…」
❌ 「ライブラリの不具合では？」

✅ 「まず自分のコードを疑おう」
```

OSやコンパイラのバグを疑いたくなる気持ちはわかる。  
でもほとんどの場合、原因は自分のコードにある。

---

## ゴムのアヒルちゃんに相談しよう🦆

### 問題を声に出して説明するだけでバグが見つかることがある

```
手順:
1. ゴムのアヒルちゃんを用意する
2. コードを1行ずつ、アヒルちゃんに向かって声に出して説明する
3. 説明しているうちに「あ、ここがおかしい」と気づく

🦆 アヒルちゃん：「…（無言）」
👨‍💻 あなた：「このメソッド、引数を受け取って処理して…
             あれ、そもそもこの引数って呼び出し元で
             初期化してたっけ？」
🦆 アヒルちゃん：「…（無言）」

```

声に出して説明することで、思い込みが外れて  
コードを改めて客観的に見られるようになる。
